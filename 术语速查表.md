# 《领域知识工程学》术语速查表

> 研读过程中随时查阅，帮助快速定位概念定义

---

## 核心术语（按章节排列）

### 第1章 理论体系

| 术语 | 核心定义 | 关键词 |
|------|---------|--------|
| **信息社会** | 以信息技术体系为载体，通过承载、传递和处理客观真实信息和知识信息，管理和影响客观世界的新型社会 | 自动化、数字形态+实体形态 |
| **社会体系** | 人类社会由主体、客体和行为通过相互作用形成的稳定、有组织且相互联系的复杂整体 | 主体+客体+行为 |
| **技术体系** | 分为非信息技术体系（农业/工业技术）和信息技术体系（计算机/网络/软件） | 双重结构 |
| **信息技术体系** | 以二进制编码的电子介质为载体，由计算机、软件和网络通信技术构成 | 替代+表达生物行为 |
| **社控系统** | 类比"工控系统"，通过对社会主体（人/组织）行为的信息化控制，实现社会自动化 | 人对人的自动化控制 |
| **工控系统** | 信息技术对技术体系（物）的自动化控制，如自动流水线、程控交换机 | 人对物的自动化控制 |
| **"人化"** | 信息技术替代和表达人的生物本能行为（肢体、感官、思维），但不能替代社会组织行为 | 生物人"消失"，社会人长存 |
| **拟量子纠缠** | 实体空间与数字空间之间类似量子纠缠的映射关系，实现效率倍增、效能倍增 | 生物↔社会、实体↔数字 |
| **价值论方法** | 内禀价值是驱动人类社会发展的源动力，实现路径是制定方案+理性行为 | 价值→动力→行为 |
| **动力机制建构** | 识别不同社会角色的价值追求，设计符合其追求的动力机制，激发内生动力 | 识别价值→设计机制→激发行为 |
| **实例化方法** | 运用历史样本总结模式，应对未来同类任务或问题 | 从经验到模式 |
| **描述性知识** | 文化、伦理、习俗等描述客观现象的知识 | 是什么 |
| **程序性知识** | 法律、标准、规范等规定行为程序的知识 | 怎么做 |

### 第2章 信息机制

| 术语 | 核心定义 | 关键词 |
|------|---------|--------|
| **感知机制** | 获取客观世界事实信息的方式和过程 | 信息采集 |
| **认知机制** | 从信息中提炼知识，包括描述性和程序性知识 | 信息→知识 |
| **控制机制** | 利用信息和知识对社会组织行为实施控制 | 知识→行为 |

### 第3章 任务范式

| 术语 | 核心定义 | 关键词 |
|------|---------|--------|
| **任务范式** | 对人类经济社会活动特定任务过程的基本模式的定义 | 本书核心理论 |
| **领域限定** | 任务必须在特定的、最小化的领域范围内定义 | 边界明确 |
| **双子系统** | 任务由主动系统（执行者）和对象系统（被作用对象）构成 | 主体+客体 |
| **问题驱动** | 以具体领域问题作为任务发起的起点和驱动力 | 问题导向 |
| **目标约束** | 目标对任务的方向、范围、标准形成限定和约束 | 结果导向 |
| **事实激活** | 用客观事实信息触发任务流程的启动和执行 | 信息触发 |
| **知识制导** | 用领域知识引导任务执行的方向和方式 | 知识引导 |
| **任务赋值** | 对任务的价值、优先级进行量化和评估 | 价值评估 |
| **数字倍增** | 通过数字化实现任务效率（速度）和效能（质量）的指数级提升 | 数字化红利 |

### 第4章 范式演化

| 术语 | 核心定义 | 关键词 |
|------|---------|--------|
| **信息能力** | 获取、处理、传递和利用信息的能力，是范式演化的驱动力 | 驱动力 |
| **本体构建** | 建立特定领域的概念模型，定义实体、属性、关系 | 概念建模 |
| **事实还原** | 从数据中还原客观世界的真实状态和变化 | 数据→事实 |
| **事件建模** | 对事件过程进行结构化表达，包括时间、地点、主体、客体等要素 | 过程建模 |
| **指标量化** | 用可度量指标对任务效果进行评价 | 可度量 |

### 第5章 工程创意

| 术语 | 核心定义 | 关键词 |
|------|---------|--------|
| **问题三维度** | 从深度、广度、价值三个维度分析问题 | 立体分析 |
| **逆向实证** | 从期望的结果出发，反向推导需要的原因和条件 | 结果→原因 |
| **微创新创意** | 在现有基础上进行小步迭代、持续改进的创新方式 | 渐进式创新 |
| **深度创新创意** | 突破性、颠覆性的创新，重新定义问题和解决方案 | 突破式创新 |

### 第6章 工程方法

| 术语 | 核心定义 | 关键词 |
|------|---------|--------|
| **产品业态** | 知识产品按形态分类，如咨询报告、软件系统、解决方案等 | 产品形态 |
| **资源预置** | 在工程启动前提前配置所需的人力、技术、资金等资源 | 前置准备 |
| **伴随服务** | 产品交付后持续提供支持、优化和升级服务 | 持续服务 |

### 第7章 领域知识工程师

| 术语 | 核心定义 | 关键词 |
|------|---------|--------|
| **领域知识工程师** | 具备跨学科、综合性和集成创新能力，从外部识别和集成专业知识解决领域问题的新型人才 | 集成者，非专家 |
| **角色化实训** | 在模拟的角色场景中进行训练，培养实战能力 | 模拟训练 |
| **业态创新** | 创造出全新的业务形态和商业模式 | 新业态 |
| **动力机制触发** | 通过识别人的价值追求，设计机制激发其内生动力的方法 | 内驱力 |

### 第8章 超级应用

| 术语 | 核心定义 | 关键词 |
|------|---------|--------|
| **超级应用** | 将领域知识工程学原理应用于数字政府、智慧城市等大型复杂场景的综合实践 | 大规模集成 |

## 容易混淆的概念辨析

| 易混概念 | 区别要点 |
|---------|---------|
| **工控系统 vs 社控系统** | 工控=人对物的控制（工业自动化）；社控=人对人的控制（社会自动化） |
| **信息技术体系 vs 非信息技术体系** | 信息技术=计算机/网络/软件；非信息技术=农业/工业等传统技术 |
| **描述性知识 vs 程序性知识** | 描述性=是什么（文化、伦理）；程序性=怎么做（法律、标准） |
| **微创新 vs 深度创新** | 微创新=小步迭代改进；深度创新=突破性颠覆 |
| **主动系统 vs 对象系统** | 主动=执行任务的主体；对象=被作用的对象 |
| **信息社会 vs 工业社会** | 信息社会=实体+数字二元形态；工业社会=仅实体形态 |
| **实例化方法 vs 案例研究** | 实例化=从样本总结模式用于未来；案例研究通常仅分析特定案例本身 |
