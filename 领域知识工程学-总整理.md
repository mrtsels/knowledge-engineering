社会自动化（核心命题）
    │  工控系统（物）→ 社控系统（人）

    ▼

    │  领域限定 + 双子系统 + 五要素 + 问题驱动 + 目标约束

工程方法 → 八种生产方法 → 产品设计/开发/运营 → 资源预置 → 伴随服务

    │  互联互通互操作 + 预置/仿真/对称/反馈 + 角色行为软件

    实体空间的任务过程以数字化方式实现效率指数级提升

## 链条二：研究方法线

```
价值论方法
    │  内禀价值 → 六种外显形态（精神：文化/法律/伦理/习俗 + 物质：发展/秩序/突发）
    │  识别"谁的价值受损/受益"→ 理解不同角色的价值追求
    │
    ▼
动力机制建构方法
    │  识别利害点 → 设计趋利避害机制 → 激发内生动力 → 外化为理性行为
    │  核心假设："趋利避害"是人的本质属性
    │
    ▼
实例化方法
    │  足够样本量的历史事实 → 结构化整理分析 → 总结模式 → 应对未来同类任务
    │  实证根基，定性+定量综合
    │
    ▼
知识创新（触发条件：已有知识体系不适用全新事件）
    │  程序性知识的立改废 + 描述性知识的基础研究回归
```

## 链条三：信息机制线

```
感知类（6种，信息的获取和存在形式）
    │
    ├── 标识 → 辨别事物（"这是谁/什么"）
    ├── 留痕 → 确保真实性（直接表征 vs 间接表达）
    ├── 公开 → 保障公平公正
    ├── 共享 → 满足行为协同需求
    ├── 汇聚 → 对冲信息分散风险，还原事实全貌
    └── 安全 → 法权保护（主权+产权+人权）
    │
    ▼
认知类（2种，信息的加工和利用）
    │
    ├── 预置 → 在信息系统中预设程序性知识和描述性知识
    └── 仿真 → 基于预置知识+事实信息，产生新知识
    │
    ▼
控制类（2种，信息的反馈和调节）
    │
    ├── 对称 → 理性决策的前提（各方信息尽可能对等）
    └── 反馈 → 结果对原因的反作用（监测↔对策闭环）
```

## 链条四：工程方法IPO线

```
输入（I）
    │
    └── 形式化搜集 → 获取领域信息资源（三大类十一小类）
    │
    ▼
处理（P）
    │
    ├── 主题化标引 → 萃取本体，建立本体库
    ├── 逻辑化存储 → 建立信息与数据的映射关系
    ├── 关联化分析 → 定性分析，建立业务模型
    └── 数量化计算 → 定量分析，建立数学模型和算法
    │
    ▼
输出（O）
    │
    ├── 结构化表达 → 确定向人和机输出的信息内容和形式
    ├── 反馈化控制 → 基于指标监测调整系统运行状态
    └── 可视化呈现 → 以图形方式呈现信息
```

## 链条五：问题解决深度线

```
现象（标）
    │  感官可感知的表象
    │  解决方案：治"标"（应急处理）
    │
    ▼
根源（本）
    │  导致现象的直接原因（物质原因/社会规则原因）
    │  解决方案：微创新创意（时间落点前置 + 空间普及）
    │
    ▼
症结（根）
    │  根源中最关键的因素，与内生动力和利益机制相关
    │  解决方案：深度创新创意（重新设计利益机制 + 动力机制）
    │  三大难点：认识难 · 调整难 · 制度化难
```

| 工控系统 | 信息技术对"物"的自动化控制（已相当成熟） |

| 数字产业 | 供给侧/需求侧微观主体行为的数字化 |

## 第2章 信息机制

| 术语 | 定义 |
|------|------|
| 社会控制 | 为实现人的内禀价值，优化经济社会发展/秩序/突发三种状态而采用的自动化、协同化、知识化控制 |
| 监测↔对策闭环 | 社会控制的基本模式：监测对象系统状态→制定对策→执行→再监测→循环 |
| 标识 | 解决客观事物辨别问题（如身份证号、RFID、二维码） |
| 留痕 | 客观事物的直接表征，具有真实性属性（**痕迹信息 vs 填报信息**是关键辨析） |
| 公开 | 不加隐蔽，与"秘密"相对，目的是实现公平公正 |
| 共享 | 基于多主体执行任务的**行为协同需求**，没有协同就没有共享需求 |
| 汇聚 | 逻辑层面的会合（非物理集中），对冲信息分散风险，还原事实全貌 |
| 安全 | 基于现实世界中各主体间的**法权边界**来确定——安全 = 主权 + 产权 + 人权 |
| 预置 | 在信息系统中预先储存程序性知识（怎么做）和描述性知识（是什么） |
| 仿真 | 基于预置知识，对模拟或真实的事实信息进行处理，得出新知识 |
| 对称 | 基于一个任务，相关多方掌握的信息尽可能对等——**理性决策的前提条件** |
| 反馈 | 控制系统把信息输送出去，又把作用结果返送回来，对信息再输出发生影响 |
| 纳什均衡 | 每个理性参与者都不会有单独改变策略的冲动（对称+博弈的结果） |
| 博弈 | 基于信息对称，通过衡量利弊的理性分析，做出最符合自身利益的决策 |

## 第3章 任务范式

| 术语 | 定义 |
|------|------|
| 领域限定 | 以最小机构的最小职能、职责事项作为划分边界 |
| 主动系统 | 施行行动、主导行动的一方（如政府监管部门） |
| 对象系统 | 主动系统的工作对象、使命对象（如被监管企业），非"被动系统" |
| 五要素 | 主体、行为、客体、时间、空间——五者一体，不可分割 |
| 主体 | 法人（营利/非营利/特别/非法人）+ 自然人 |
| 行为 | 生物本能行为（肢体/感官/思维）+ 社会组织行为（法人行为），两者耦合纠缠 |
| 客体 | 主体为解决问题而利用和产生的物质和非物质资源 |
| 三类角色 | 领导（决策/协调/指挥）、办公室/信息保障部门（信息服务）、职能部门（监测/执行/监督） |
| 问题驱动 | 问题是推动两系统运行的动力，内驱力产生于理想与现状的差距 |
| 问题三层结构 | 现象（表象）→ 根源（直接原因）→ 症结（最关键因素，与利益格局相关） |
| 目标约束 | 目标确定问题范围和解决程度，约束两系统五要素的资源配置 |
| 事实激活 | 特定事实信息激活特定数字系统运转（如超速事实→自动扣分） |
| 知识制导 | 描述性知识+程序性知识引导行为向目标接近 |
| 数字倍增 | 实体空间任务过程以数字化方式实现效率倍增、时空协同和自动控制 |
| 五种指标值 | 理想值、现状值、问题值、目标值、对标值 |
| 实体空间 | 人类经济社会活动开展的现实世界 |
| 数字空间 | 实现社控系统自动化、协同化、知识化的信息系统/网络空间 |

## 第4章 范式演化

| 术语 | 定义 |
|------|------|
| 信息能力 | 一个任务范式实例中所有角色解决问题/实现目标行为的信息输入、处理、输出总体能力 |
| IPO模型 | Input→Process→Output：信息在信息系统中以输入→处理→输出闭环方式流动 |
| 流程化处理 | 直接引用程序性知识对事实信息进行判定 |
| 知识化处理 | 机器调取知识信息处理事实信息，产生新知识（替代人的思维行为） |
| 本体（Ontology） | 两系统的系统行为过程的知识体系，是"共享概念模型的形式化规范说明" |
| 本体构成 | 概念（类）+ 属性（范式属性+信息属性）+ 关系（因果关系为核心） |
| 本体网 | 由本体节点（概念）和边（关系）构成的网状结构 |
| 本体构建原则 | 分类分级、相关相连、走向实例 |
| 人工标引 vs 机器自动标引 | 人工标注+技术挖掘规律→自动化，两者交叉交替进行 |
| 还原性 | 本体是否具有还原为事实的属性 |
| 还原度 | 事实信息对事实的还原程度：痕迹信息（最高）> 全量信息 > 样本信息 > 填报信息（最低） |
| 发展事件 | 解决资源配置的效率和公平问题（合理性） |
| 秩序事件 | 解决社会合规性问题（规范与有序） |
| 突发事件 | 自然灾害/公共卫生/社会安全/事故灾难类，四个等级（I-IV级） |
| 通用模型 vs 专业模型 vs 应用模型 | 无差别模型 vs 领域专业模型 vs 具体问题的模型 |
| 绝对指标 | 总量指标，反映总体规模或水平 |
| 相对指标（六种） | 计划完成程度/结构/比例/比较/强度/动态 |

## 第5章 工程创意

| 术语 | 定义 |
|------|------|
| 价值取向 | 宏观层面（社会价值观）+ 微观层面（具体领域政策法规） |
| 负价值 | 与宏观社会价值取向相反的状态，即"问题" |
| 问题值 | 问题值 = 理想值 - 现状值 |
| 现象 | 事物表现出来的，能被人感觉到的一切情况（"标"） |
| 根源 | 问题产生的直接原因，分直接物质原因和间接社会/规则原因（"本"） |
| 症结 | 根源中最关键、起最重要作用的因素，与内生动力和利益机制相关（"根"） |
| 逆向实证 | 向历史要未来——从同类问题的历史事件中找到规律，激发创意 |
| 拼叠研究 | 搜集同类事件样本，拼叠事件的属性，合并出覆盖未来事件的属性 |

| 深度创新创意 | 针对症结的创意——重新设计利益机制和动力机制，激发内生动力 |

| 实体创新 | 在实体空间找到解决问题的方案创新（微创新或深度创新） |

## B. 简答题（第11-20题）

**Q11**：请用一句话概括"领域知识工程学"的定义。

> **参考答案**：领域知识工程学是与"新工科""新文科"一致的原创学科，以信息社会人类经济社会活动特定任务过程为研究对象，研究并设计特定任务问题的解决方案及其自动化、协同化、知识化工程，它的研究对象是任务范式、范式实例、实例的自动化及自动化实例的普及率。

---

**Q12**：请解释什么是"双子系统"，并分别举一个政府范畴和产业范畴的例子。

> **参考答案**：双子系统指人类经济社会活动总是以主动系统（施行行动方）和对象系统（工作对象/使命对象）两系统强相关互作用的方式展开。
> - 政府范畴：主动系统=食品安全监管部门，对象系统=学校食堂/食品生产企业
> - 产业范畴：主动系统=产品供应商（供给侧），对象系统=消费者/客户（需求侧）

---

**Q13**：简述领域知识工程学"价值论→动力机制→实例化"三大研究方法之间的关系。

> **参考答案**：价值论方法识别"谁的价值受损/受益"（基础），动力机制建构方法通过识别利害点设计趋利避害机制激发内生动力（核心），实例化方法从历史样本中总结模式提供实证根基（验证）。三者递进：先理解价值追求→再设计驱动机制→最后用实证验证和优化。

---

**Q14**：什么是"逆向实证"？为什么说"未来世界可能发生的事件与已发生事件的重合率很高"？

> **参考答案**：逆向实证是"向历史要未来，从过去找出路"——搜集足够样本量的同类问题历史事件，还原每个事件的问题三维度、处理过程、结果，找到规律和特点，激发创意灵感。重合率高的原因是：人类社会活动的基本模式（任务范式）是相对稳定的——领域、双子系统、五要素的基本结构不变，特定领域的同类问题在根源和症结层面高度相似。

---

**Q15**：请说明"制度安排"在领域知识工程学中的必要性，以及不做制度安排的后果。

> **参考答案**：必要性：工程创意的核心（时间落点前置/空间普及或重新设计利益机制）本质上是调整或打破现有规则，必须通过制度安排固化为具有约束力的规章准则。不做制度安排的后果：系统变成"假大空"系统——信息源无法保障（无人有义务提供数据）、运营使用无法落实（无人有义务按系统要求改变行为）、自动化无法展开（无人遵循无依据的规则）。

---

**Q16**：八种生产方法之间是什么关系？它们如何形成完整的生产流程？

> **参考答案**：八种生产方法按IPO模型形成不可逆的工艺流程链：输入（形式化搜集）→处理阶段四步（主题化标引→逻辑化存储→关联化分析→数量化计算）→输出阶段三步（结构化表达→反馈化控制→可视化呈现）。前一步的输出是后一步的输入，类似于汽车生产线的冲压→焊接→涂装→总装流程。

---

**Q17**：领域知识工程学的人才培养中，教师的"三导"是什么？各自侧重点有什么不同？

> **参考答案**：引导=引导学生自我完成条件性知识的获取和学习（方向指引），指导=将领域知识工程学的任务范式模式对学生进行植入（方法指导），制导=强制性约束和敦促学生反复实践工程方法，形成动力定型和条件反射（强制重复）。三者递进：引导给方向→指导授方法→制导促固化。

---

**Q18**：超级应用的四大核心功能是什么？它们各自对应COVID-19疫情防控中的什么操作？

> **参考答案**：1. 自动寻迹——通过手机轨迹回溯确诊者的密切接触者（对应"早发现"）；2. 自动禁足——通过利害信息推送控制确诊者和密切接触者的活动范围（对应"早隔离"）；3. 自动资源配置——根据仿真计算最优的医疗资源（人员/防护服/口罩/检测盒）配置（对应"早治疗"的资源保障）；4. 自动奖惩兑现——根据预置奖惩规则实时判定行为合规性并兑现（对应"早报告"的动力机制保障）。

---

**Q19**：什么是"资源预置"？为什么领域知识工程学能做到资源预置而传统IT做不到？

> **参考答案**：资源预置是在需求出现之前就预先制作好产品的组成部分（信息构件+功能组件）并存储放置，待具体产品需要时组合取用。能做到的原因是领域知识工程学具有确定性和高度开放性——领域、任务、目标、条件、用户角色及角色行为都是确定的，因此可以预先生产。传统IT需要等到拿到具体项目后才能确定用户需求，无法预置。类比：领域知识工程产品似"活字印刷"（预置字块组合），传统IT似"雕版印刷"（完整定制）。

**Q20**：请用"领域限定→双子系统→问题驱动→目标约束→事实激活→知识制导"这六个概念，描述一个具体任务的完整范式框架（可自选案例）。

> **参考答案（以电梯安全监管为例）**：领域限定=电梯维保与使用安全的政府监管职能；双子系统=主动系统（市场监管局）+ 对象系统（电梯维保公司/使用单位）；问题驱动=电梯坠落事故→现象（事故伤亡）→根源（制动器维保不到位）→症结（维保公司追求利润最大化、监管信息不对称）；目标约束=将电梯坠落事故率降至零，约束范围限于曳引驱动电梯的制动器维保环节；事实激活=电梯维保时间记录、制动器状态检测数据进入系统→激活维保质量判定→关联维保公司和检验人员信息；知识制导=预置《特种设备安全法》维保标准（程序性知识）+ 制动器工作原理（描述性知识）→指导维保行为的自动化监测和判定。

## C. 深度思考题（第21-28题）

**Q21**：本书的核心批判之一是"唯技术论"。请结合全书内容，论述什么是"唯技术论"，领域知识工程学如何避免它？

> - 唯技术论的表现：过度关注技术本身（云计算、大数据、AI、区块链），以为有了技术就能解决问题，忽略学科能力建设和行为方式重构
> - 领域知识工程学的替代方案：
>   1. 需求先于技术（问题驱动→目标约束→实体创新→数字倍增）
>   2. 实体创新先于数字化（先有解决方案再数字化，否则是"低效的电子版"）
>   3. 制度安排先于系统开发（先有制度保障再有技术实现）
>   4. 人才先行（先有领域知识工程师再有工程实施）
>   5. 信息机制（预置/对称/反馈等）比信息技术（用什么数据库/编程语言）更重要
>   6. 互联互通互操作≠建大系统——利用已有系统而非推倒重来
> - 本质：领域知识工程学认为，技术是手段而非目的，技术必须服务于实体空间的创新解决方案，且必须在制度和人才的保障下才能发挥真正作用。

**Q22**：请对比领域知识工程学的"实体创新+数字倍增"与当前流行的"数字化转型"，分析两者的异同。

> **参考答案框架**：
> - 相同点：都追求通过数字化提升效率
> - 不同点：
>   1. 起点不同：数字化转型常以"有什么技术"为起点，领域知识工程以"有什么问题"为起点
>   2. 路径不同：数字化转型常先建IT系统再适配业务，领域知识工程先实体创新再数字化
>   3. 深度不同：数字化转型多停留在"流程电子化"，领域知识工程追求"社会自动化"（人的社会组织行为数字化）
>   4. 制度维度：数字化转型很少涉及制度安排，领域知识工程将制度安排视为必要条件
>   5. 人才维度：数字化转型依赖IT工程师，领域知识工程需要新型的领域知识工程师
>   6. 确定性：数字化转型往往"边做边看"，领域知识工程强调"五确定"
> - 结论：领域知识工程学为数字化转型提供了一套更系统、更深入的方法论，可以理解为"数字化转型的工程化、学科化升级版"。

**Q23**：第8章的COVID-19超级应用构想在技术上并不复杂（5G富媒体短消息 + 知识库 + 手机轨迹），但现实中未能实现。请从制度、组织、利益格局角度分析阻碍因素。

> **参考答案框架**：
> 1. 制度层面：信息直达中央的制度安排与现行层级上报体制存在冲突；各部门数据互通的法规壁垒（如电信运营商数据、公安数据、医疗数据的跨部门调用权限）
> 2. 组织层面：缺乏一个能协调所有相关部门的顶层组织；各级政府对"一例即启动"的谨慎态度（担心"误报"的政绩后果）
> 3. 利益格局：奖惩前置可能改变现有权力分配（中层官员的权力空间被压缩）；"自动奖惩兑现"可能触发对"机器决策"的信任危机
> 4. 社会认知：公众对"全量、在线、实时"的数据收集可能存在隐私担忧
> 5. 本书的答案：根本原因是缺乏领域知识工程师这个角色来"顶层横贯"——现有的治理结构中没有一个角色能同时理解疫情防控的医学知识、信息技术的工程方法、行政管理的制度逻辑，并将其集成为一个可运行的社控系统。

**Q25**：领域知识工程学强调从问题三层结构（现象→根源→症结）的深度分析来设计解决方案。请选择一个你熟悉的社会问题，尝试用这个框架进行分析，并给出微创新和深度创新的建议。

> **参考答案框架（以交通拥堵问题为例）**：
> - 现象：城市核心区早晚高峰严重拥堵，通勤时间倍增
> - 根源：出行需求在时空上的高度集中（上下班时间一致、核心区就业岗位集中）+ 道路容量不足 + 公共交通吸引力不够
> - 症结：城市规划中的"职住分离"利益格局——核心区土地价值高→开发商追求商业办公用地→居住功能外迁→通勤距离拉大→道路难以扩容→拥堵日益严重
> - 微创新建议：错峰上下班制度（时间落点前置），在全市范围内普及弹性工作制（空间普及）
> - 深度创新建议：改革城市规划的用地结构——在核心区强制配建一定比例的住宅（改变土地利益分配机制），同时将道路容量、通勤时间纳入土地出让的前置评估条件（制度创新）
