# 《领域知识工程学》预习指南

## 一、这是一本什么书？

《领域知识工程学》是一本原创专著，开创一门新学科。其核心任务是回答四个根本问题：

| 问题 | 答案 |
|------|------|
| 人类社会运作的基本模式是什么？ | **任务范式** — 一个结构化的理论框架 |
| 信息社会的本质是什么？ | **拟量子纠缠** — 实体空间与数字空间的纠缠、映射 |
| 未来社会新生产力特征是什么？ | **社会自动化** |
| 未来新兴劳动者是谁？ | **领域知识工程师** |

**学科定位：** 本体论、认识论、方法论相统一的新学科。它不进入专业知识的内部，而是从外部识别、集成解决特定领域问题所涉及的专业知识方法。

## 二、全书逻辑脉络（"为什么这样编排"）

### 总纲
本书的逻辑是：**理论 → 机制 → 范式 → 演化 → 创意 → 方法 → 人才 → 应用**

```
第1章 理论体系 ──→ 奠定学科基础（是什么、为什么）
       ↓
第2章 信息机制 ──→ 理解社会如何通过信息被控制（机制原理）
       ↓
第3章 任务范式 ──→ 核心理论框架（社会运作的基本模式）
       ↓
第4章 范式演化 ──→ 范式如何随信息能力提升而演化（动态发展）
       ↓
第5章 工程创意 ──→ 如何发现问题、产生创意（创新方法论）
       ↓
第6章 工程方法 ──→ 如何把创意变成产品（工程化实施）
       ↓
第7章 领域知识工程师 ──→ 谁来执行？（新人才角色）
       ↓
第8章 超级应用 ──→ 综合案例与前景展望
```

## 三、核心概念的"预热"（研读前要先了解）

### 1. 三大社会形态
- **农业社会** — 以土地为核心，生产力低，依赖体力
- **工业社会** — 以机器为核心，生产力指数级提升，工业自动化
- **信息社会** — 以信息技术为核心，实体与数字空间"纠缠"，追求社会自动化

### 2. 信息技术体系 vs 非信息技术体系
- **非信息技术体系**：农业技术、工业技术等传统技术体系
- **信息技术体系**：计算机、网络通信、软件 — 作用是"替代和表达"生物行为，放大非信息技术的效率

### 3. 工控系统 vs 社控系统（全书最核心的概念之一）
- **工控系统**：信息技术对技术体系（物）的控制 → 工业自动化
- **社控系统**：信息技术对社会体系（人+组织）的控制 → 社会自动化
- 当前OA、审批等系统只是低层次的社控系统，缺失真实性和知识性

### 4. 拟量子纠缠
书中的核心譬喻，用来描述信息社会特有的现象：
- 生物行为 ↔ 社会组织行为 的纠缠
- 生物行为 ↔ 信息技术 的纠缠
- 客观事实信息 ↔ 数据载体 的纠缠

### 5. 任务范式
这是全书核心理论，包含几个关键要素：
- **领域限定**：特定的、最小化的领域范围
- **双子系统**：主动系统（人/组织） + 对象系统（被作用的对象）
- **问题驱动**：以具体问题为出发点
- **目标约束**：目标对任务有约束力
- **事实激活**：用事实信息激活任务
- **知识制导**：用知识引导任务执行
- **数字倍增**：数字化带来效率指数级提升

## 四、研读策略建议

### 第一天：打地基（第1-3章）
- **第1章**是全书总纲，读懂了这一章，后面就顺了
- **第2章**偏机制原理，理解"信息如何控制社会"
- **第3章**是全书理论核心，需要重点理解"任务范式"的9个要素
- **重点关注**：社控系统、任务范式、双子系统

### 第二天：上台阶（第4-6章）
- **第4章**讲范式如何演化（本体构建、事件建模、指标量化）
- **第5章**讲创新创意方法论（问题三维度、逆向实证、微创新、深度创新）
- **第6章**篇幅较长，讲工程化方法（产品业态、生产方法、工程阶段）
- **重点关注**：三种创新方法、工程化生产流程

### 第三天：见全貌（第7-8章+总复习）
- **第7章**篇幅最长，讲领域知识工程师的培养与实训
- **第8章**综合应用案例
- **重点关注**：如何将前两天的理论应用于实践

## 五、心态准备

1. **这是一本跨学科著作** — 融合哲学、逻辑学、语言学、社会学、数学、计算机科学
2. **作者有深厚的工程实践背景** — 不是纯学术著作，而是来自国家信息化重大工程的实践经验总结
3. **读法建议** — 先抓框架再填细节，每个概念问自己："这个概念在解决什么问题？"
4. **约450页**，信息量大，三天时间紧凑，保持专注
