## 第1章 理论体系 · 概念树

```
1.1 时代潮流
  ├── 信息社会（继农业、工业社会后的新阶段）
  │     ├── 特征1：对客观世界行为的控制
  │     └── 特征2：促进客观世界知识的转化
  ├── 社会体系
  │     ├── 主体（自然人+组织）
  │     ├── 客体（技术体系：非信息技术+信息技术）
  │     └── 行为（人对物 + 人对人）
  └── 社控系统
        ├── 类比：工控系统 → 工业自动化
        ├── 新概念：社控系统 → 社会自动化
        ├── 误区1：流程论（流程容易改，不是本质）
        └── 误区2：技术论（技术往往是被动的）

1.2 学科渊源
  ├── 新工科（人工智能、智能制造等）
  ├── 新文科（文理交叉、跨学科）
  └── 领域知识工程学 = 扬弃式承续新工科+新文科

1.3 研究对象
  └── 任务范式、范式的实例、实例的自动化、自动化普及率

1.4 研究方法
  ├── 价值论方法（内禀价值是驱动源动力）
  ├── 动力机制建构（识别角色价值追求→设计机制→激发行为）
  └── 实例化方法（从历史样本中总结模式应对未来）

1.5 知识体系
  ├── 本学科知识
  ├── 条件性知识及获取方式
  └── 集成多专业知识

1.6 学科特点
  ├── 确定性（领域/任务/因果/条件/目标确定）
  └── 工程化（知识产品结果+生产过程=工程化）

1.7 学科应用
```

## 全书核心概念关系图

```
信息社会 (时代背景)
    │
    ▼
社控系统 (核心诉求)
    │
    ├── 需要 → 任务范式 (理论框架)
    │               │
    │               ├── 领域限定 + 双子系统 (结构)
    │               ├── 问题驱动 → 目标约束 (动力)
    │               ├── 事实激活 + 知识制导 (执行)
    │               └── 任务赋值 → 数字倍增 (效果)
    │
    ├── 依赖 → 信息机制 (感知→认知→控制)
    │
    ├── 演化 → 范式演化 (本体→事件→指标)
    │
    ├── 产生 → 工程创意 (微创新→深度创新)
    │
    ├── 落地 → 工程方法 (需求→设计→开发→运营)
    │
    ├── 执行者 → 领域知识工程师 (人才培养)
    │
    └── 目标 → 超级应用 (智慧城市/数字政府...)
```
